    Introduction

    Welcome to the noblox.js API! This site will help you get started with noblox.js. Once you're familiar, you should be fine to navigate this website yourself.

    Authentication

    To log into a Roblox account, use this code:

    const noblox = require('noblox.js');

async function run() {
  await noblox.cookieLogin('_ROBLOSECURITYKEYFORCOOLBOY57');
}

run();
    The following code utilizes the async/await feature in ES8. Be sure you're running the latest node version and latest version of noblox.js.

    Make sure to replace _ROBLOSECURITYKEYFORCOOLBOY57 with your .ROBLOSECURITY key. You can find this by heading over to a Roblox page and signing in. Once you're in, open Inspect Element in your browser (Control + Shift + I). Click the Application Tab and find the .ROBLOSECURITY line and copy its contents, starting with _|WARNING:-DO. Put the full token, including the warning, into the cookieLogin line.

    Following Roblox's recent Funcaptcha update, users can no longer log into their accounts through noblox.js using the basic username/password combination. In order to get around this, noblox.js bypasses this and uses your .ROBLOSECURITY key to login. Your .ROBLOSECURITY key doesn't go anywhere apart from your local computer and to Roblox directly so you can trust the process.

    Promises

    Example

    nbx.getPlayerInfo(44545891).then((playerInfo) => {
  console.log("This user's status is " + playerInfo.status)
}).catch(console.error)

    If you wish to take a shortcut, you may use async/await which allows you to do the following:

    async function exec() {
  const userId = await nbx.getIdFromUsername('Qxest') // 55549140

  console.log("This user's ID is " + userId)
}

    The biggest issue and the most frequently asked question is always along the lines of "why isn't this working" and the answer is because you're not using promises properly. Every noblox.js function returns a Promise. You can read more about Promises here. In a nutshell, when you use a noblox.js function you must treat it as a promise as shown in this example.

    Errors

    The Roblox API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- You are not allowed to do this.
    403 Forbidden -- The requested content is not available to you. (Likely not logged in, lack group permissions, bad token)
    404 Not Found -- The specified content could not be found.
    405 Method Not Allowed -- You tried to access an endpoint with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    429 Too Many Requests -- You're sending too many requests! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
